# Reassign director package owner

Админский ручной перенос одного пакета e-Qazyna по ФИО учредителя/руководителя.

## Что делает workflow

Workflow `Reassign director package owner` принимает ФИО и ID нового пользователя Bitrix. После этого он переносит на нового ответственного:

- все найденные e-Qazyna компании этого руководителя/учредителя;
- все e-Qazyna сделки выбранных компаний;
- сделки, где руководитель найден в комментариях сделки или через контакт;
- связанные контакты выбранных компаний/сделок;
- контакты, у которых ФИО совпадает с введённым руководителем.

Старый ответственный не указывается. Скрипт переносит пакет независимо от того, на ком сейчас висят карточки.

## Безопасный порядок запуска

1. Открыть **Actions → Reassign director package owner → Run workflow**.
2. Заполнить:
   - `director_name` — ФИО, минимум фамилия + имя/инициалы, например `ЛЯБАХ Г.Г.`;
   - `target_user_id` — ID нового ответственного пользователя;
   - `dry_run = true`.
3. Запустить и скачать артефакт `reassign-director-package-owner-log`.
4. Проверить CSV: компании, сделки, контакты, текущие владельцы, будущий владелец.
5. Если список верный, повторить запуск с `dry_run = false`.

## Важные ограничения

- Один токен вроде `ЛЯБАХ` запрещён: это слишком рискованно для массового переноса.
- По умолчанию переносятся только сущности `ORIGINATOR_ID = EQAZYNA`.
- По умолчанию закрытые/проваленные сделки тоже входят в перенос, потому что это ручной перенос всего пакета.
- `target_user_id` не обязан быть в `eqazyna_bitrix/config/managers.yml`. Скрипт проверяет его через Bitrix `user.get`: пользователь должен существовать в Bitrix и быть активным.
- Технические пользователи из `source_responsible_ids` не могут быть целевым ответственным.

## Что workflow не делает

- Не меняет стадию сделки.
- Не меняет причину проигрыша.
- Не участвует в автоматическом распределении новых заявок.
- Не использует старые ручные закрепления по BIN/руководителям.
